Basic Spike GeneratorÂ¶

Within the Nx SDK, we provide you with a basic spike generator. The code snippet below provides some guidance on how to use it.

# Generate an input spike at timestep 5 to chip 0, core 4 and axon 0
bs       = BasicSpikeGenerator(board)
timestep = 5
chipId   = 0
coreId   = 4
axonId   = 0
bs.addSpike(timestep, chipId, coreId, axonId)

sendSpikeInputs(numSteps, currentTimeStep)Â¶

Inputs are expected to be sorted before this method is invoked

Packing protocol is [<num_messages>,<message1_time>,<message1_chip>,<message1_core>,<message1_axon_high>,<message1_axon_lo>] 3 spike messages are packed together in this protocol in a single message. Negative num_messages or time means to wait until that many steps to check again without blocking

Parameters

  • inputs (list(list)) -- each entry of list is a list of spike messages (time, chip, core, axon_high, axon_low) for that input generator. These entries are supposed to be ordered.

  • numSteps (int) -- NumSteps denotes the maxtimestep till which this run will execute. Passed as (self.currTimeStep + numSteps) in superhost

Returns

None
