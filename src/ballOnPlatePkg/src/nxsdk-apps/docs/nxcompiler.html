NxCompiler¶

Overview¶

The NxSDK compiler maps an NxNet network or network tree into the appropriate backend, e.g. Loihi hardware. The compiler is run as part of net.run(), where net is an NxNet object. The compiler is composed of two main components, the partitioner and the mapper.

The partitioner takes the NxNet network or network tree and assigns logical core IDs to all compartments in the network or network tree. The mapper then consumes the output of the partitioner (i.e. an NxNet network or network tree enhanced with logical core ID assignments) and maps it down to the NxCore interface.

Partitioner¶

The partitioner runs a partitioning algorithm on each network in a network tree. For a given network in a network tree, if a custom partitioner is specified, then the partitioner will use the custom partitioner module for that network. Otherwise, the partitioner will utilize a default partitioner module.

The requirement on any partition of a network tree is that each network in a network tree must use a set of logical core IDs that is disjoint from the set of logical core IDs used by any other network in the tree.

A custom partitioner is specified for a given network as follows:

net = nx.NxNet()
customPartitionerModule = YourPartitionerModule()
net.customPartitioner = customPartitionerModule

The custom partitioner module must be a subclass of AbstractPartitionerModule.

class nxsdk.compiler.nxsdkcompiler.partitioner_module_abstract_class.AbstractPartitionerModule¶

Abstract class for a partitioner module for network partitioning. The partitioner module will partition a single network, not a network tree. It will be used with the network to which the module is attached.

abstract partition(network, availableCoreIds)¶

Partitions the NxNet network by assigning logical core IDs to all compartments.

The partition method is responsible for utilizing logical core IDs from the availableCoreIds and removing those logical core IDs from the set. A logical core ID should be removed from availableCoreIds as soon as any compartment is assigned the logical core ID, i.e. the logical core IDs in availableCoreIds contain only unassigned logical core IDs.

Depending on the network features being used, different restrictions need to be considered. For example:

-- Compartments: Absent configurations such as delay, the basic restriction is that a logical core can generally have no more than 1024 compartments.

-- Neurons: All compartments associated with a given neuron need to be on the same logical core.

-- Shared connections (advanced): Compartment groups involved in shared connections must be partitioned to conform with HW requirements.

If network features such as neurons or shared connections are not supported, please utilize the validateNetwork method to check for unsupported features.

Parameters

  • network (NxNet) -- An NxNet object

  • availableCoreIds (set(int)) -- The set of unused logical core IDs

Returns

The updated set of unused logical core IDs

Mapper¶

The mapper runs a mapping algorithm on each network in a network tree. For a given network in a network tree, if a custom mapper is specified, then the mapper will use the custom mapper module for that network. Otherwise, the mapper will utilize a default mapper module.

A custom mapper is specified for a given network as follows:

net = nx.NxNet()
mapperModule = YourMapperModule()
net.customMapper = mapperModule

The custom mapper module must be a subclass of AbstractMapperModule.

class nxsdk.compiler.nxsdkcompiler.mapper_module_abstract_class.AbstractMapperModule¶

Abstract class for a mapper module for network mapping. The mapper module takes an NxNet object where all compartments have been assigned logical cores by the partitioner module and maps it into the NxCore interface. The mapper module will map a single network, not a network tree. It will be used with the network to which the module is attached.

The module interface provides for a two step mapping process. First, each network in a network tree will be be mapped using its associated mapper module's "map" method. Second, all networks will be processed using its associated mapper module's "updateMap" method. At the conclusion of the second step, the expectation is that the board is ready to be run.

In order to support the composition of multiple NxNet objects, the mapper must support mapping of OutputStubs and InputStubs to OutputMapperStubs and InputMapperStubs, respectively.

In order to connect an NxNet network that has a custom mapper module with another NxNet network, each custom mapper module must be able to support the following:

  • Implement YourOutputMapperStub that inherits from OutputMapperStubInterface.

  • Map NxNet InputStub objects to corresponding InputMapperStub objects. When an InputMapperStub object is created, call its attemptConnection method.

  • Map NxNet OutputStub objects to corresponding YourOutputMapperStub objects. When a YourOutputMapperStub is created, call its attemptConnection method.

class nxsdk.compiler.nxsdkcompiler.mapper_stubs.OutputMapperStubInterface(refOutputStub: nxsdk.net.nodes.connection_stubs.OutputStub)¶

The abstract base class for output mapper stub objects needed by the mapper to create cross network connections.

attemptConnection() → bool¶

Make the following traversal to determine if the InputMapperStubs on the other end of the connection exists:

self --> OutputStub --> [InputStubs] --> [InputMapperStubs]

If [InputMapperStubs] exists, use the parameters in that object to configure the output axon by calling self.createConnection()

Returns

True if all destination stub connections are established, False otherwise

property connEstablished¶

True if connections to input axons associated with all input mapper stubs pointed to by this output mapper stub have been established, else False.

abstract createConnection(inputMapStubLst: List[nxsdk.compiler.nxsdkcompiler.mapper_stubs.InputMapperStub]) → None¶

If possible, create the output axons pointing to the destination input axons specified in each InputMapperStub in the list inputMapStubLst. Note that at the NxNet level, each InputStub corresponds to a different NxNet object. There is then a one-to-one correspondence from an InputStub to an InputMapperStub.

Requirements: -- Set inputMapStub.connEstablished = True when an output axon to the input axons specified by a inputMapStub object in inputMapStubLst is configured. -- Set self.connEstablished = True when connections to ALL input axons specified by inputMapStubs in inputMapStubLst are established.

Parameters

inputMapStubLst (list) -- The the list of InputMapperStub objects with destination input axon information.

Returns

None

class nxsdk.compiler.nxsdkcompiler.mapper_stubs.InputMapperStub(refInputStub: nxsdk.net.nodes.connection_stubs.InputStub)¶

The mapper level class associated with an NxNet InputStub. A mapper must map an InputStub into an InputMapperStub.

A custom mapper must map an InputStub to an InputMapperStub.

attemptConnection() → bool¶

Make the following traversal to determine if the OutputMapperStub on the other end of the connection exists:

self --> InputStub --> OutputStub --> OutputMapperStub

If the OutputMapperStub exists, ask that object to configure the source HW core output axon. The OutputMapperStub will set the connection state for the InputMapperStub.
