class nxsdk.arch.n2a.graph.nodesets.stdp_post_state_obj_inf.StdpPostStateObjInf(yspike0=0, yspike1=0, yspike2=0, yepoch0=0, yepoch1=0, yepoch2=0, tspike=0, traceProfile=0, stdpProfile=0)Â¶

Represents StdpPostState NodeSet which contains the dynamic and static configuration state of synaptic post traces associated for each compartment per core. For every compartment there exist three independent synaptic post traces y[i] that each use a set of shared TraceCfg settings specified in StdpPostCfg. StdpPostState also allows to control the assignment of learning rules to the input synapses of a compartment. The learning rule for each synapse is selected according to the sum \(stdpProfile=stdpProfile1 + stdpProfile2 + stdpProfile3\) of pointers into the StdpProfile NodeSet. Here, stdpProfile3 is given by StdpPostState which allows to control the learning rule assignment by destination compartment.

property traceProfileÂ¶

(uint2) Specifies how the different TraceCfg entries in StdpPostCfg get assigned to the y1, y2 and y3 synaptic post traces. Note: This is not the same as for The traceCfgSelect parameters for synaptic pre- or reward traces that directly select a specific TraceCfg entry in StdpPreCfg. The following assignments specify which trace y[i] gets assigned to which StdpPostCfg[j] entry:

  • 0: y[0]->StdpPostCfg[3], y[1]->StdpPostCfg[1], y[2]->StdpPostCfg[2]

  • 1: y[0]->StdpPostCfg[0], y[1]->StdpPostCfg[3], y[2]->StdpPostCfg[2]

  • 2: y[0]->StdpPostCfg[0], y[1]->StdpPostCfg[1], y[2]->StdpPostCfg[3]

  • 3: y[0]->StdpPostCfg[0], y[1]->StdpPostCfg[1], y[2]->StdpPostCfg[2]
