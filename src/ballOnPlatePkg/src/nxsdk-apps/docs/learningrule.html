class nxsdk.net.nodes.learning_rule.LearningRule(id, dw=None, dd=None, dt=None, x1Impulse=0, x1TimeConstant=0, x2Impulse=0, x2TimeConstant=0, y1Impulse=0, y1TimeConstant=0, y2Impulse=0, y2TimeConstant=0, y3Impulse=0, y3TimeConstant=0, r1Impulse=0, r1TimeConstant=0, tEpoch=2, combineLiterals=True, printDebug=False)Â¶

A LearningRule can be assigned to a Synapse and describes how its three dynamic state variables (weight, delay and tag) evolve when learning is enabled.

A LearningRule is applied to a synapse at the end of a learning epoch. Each learning epoch is of duration tEpoch.

A LearningRule consists of up to three equations that specify the dynamics of the weight, delay and tag state as a function of synaptic pre-, post- and reward-traces as well as other synaptic state variables. These equations must be provided as a string and must follow a specific syntax such that it can be interpreted and compiled. When reward-traces are utilized in a learning rule, each connection associated with that learning rule will have an additional reinforcement channel associated with it. That reinforcement channel will be used for injecting reward spikes.

Note

Each dynamics equation for the weight (dw), delay (dd) and tag (dt) must be provided as a sum of products. Each product is separated by a +, or - operator while factors of a product are separated by *. A product consists of four possible types of factors: dependencies, variables, expressions and literals. Of these factors only the dependency factor is mandatory and it determines whether a product needs to be evaluated or not.

Note

Due to a bug in the HW implementation, pre- and reward-traces may sometimes register a spike sent to them with a one time step delay.

The following is a list of valid symbols permitted in a dynamics equation:

Operators: +, -, *

  • Operators + and - separate products. To negate the first product in a series of products preceed it by -. A + in front of the first product is optional and will be ignored.

  • * separates factors in a product.

Factors:
  • Dependencies: x0, y0, r0, uk

    • To avoid unnecessary evaluations of products, products must contain at least one dependency factor which conditions the evaluation of the product on that factor. x0, y0, and r0 make the evaluation of a product dependent on the presence of a pre-, post- or reward spike during the past learning epoch while uk declares a product to be evaluated unconditionally every 2**k learning epochs regardless of pre-, post- or reward spikes.

    • Any subsequent dependency factor (from left to right) will be interpreted as a multiplicative variable without affecting product evaluation.

    • r0 is not currently supported in a learning rule.

  • Variables: x[0,1,2], y[0,1,2,3], r[0,1], w, d, t

    • All x[i], y[i] and r[q] are pre-, post- or reward spike trace variables for i>0 or spike indicator variables for i==0. w, d and t represent three a synapse's weight, delay or tag state variables.

  • Expressions:
    • BracketExpression: (v+C)

      v can be any variable from above and C is a signed constant 8 bit literal constant. For v=w and v=t, there is currently a limitation on C. w and t can be in the range [-256, 256) while C must be in the range [-128, 128). To use a C value outside [-128, 128) but within [-256, 256), please avoid bracket expressions and expand out the terms. For example, K*(w + 200)*y0 could be written as K*w*y0 + K*200*y0, where K is some constant. This expansion of the bracket expression should be functionally identical, but will execute slightly slower on Loihi.

    • FunctionExpression: sgn(v+C)

      Similar to BracketExpression, just with sgn(..) function applied to it. v can only be variable from w, d and t.

  • Literal:

    Literal numeric values (i.e. scaling constants) can be specified either as a simple integer values or in a mantissa exponent representation:

    • [+,-]mantissa*2^[+,-]exp

    • [+,-]2^[+,-]exp

    • [+.-]mantissa

    The first literal in an equation is special in that only it can be interpreted explicitly in the mantissa/exponent representation. All subsequent literals will be converted into a single 7+1 bit integer.

Learning rule examples with pre-synaptic and post-synaptic traces:

lr = net.createLearningRule(dw='2*x1*y0-2*y1*x0',
                            x1Impulse=40,
                            x1TimeConstant=4,
                            y1Impulse=40,
                            y1TimeConstant=4,
                            tEpoch=2)
connProto = nx.ConnectionPrototype(weight=50, delay=0, enableLearning=1,
                                   learningRule=lr)

Learning rule with pre-synaptic, post-synaptic, and reward traces:

lr = net.createLearningRule(dt='2*x1*y0-2*y1*x0',
                            dw='u0*r1*t',
                            x1Impulse=40,
                            x1TimeConstant=4,
                            y1Impulse=40,
                            y1TimeConstant=4,
                            r1Impulse=30,
                            r1TimeConstant=2,
                            tEpoch=2)
connProto = nx.ConnectionPrototype(weight=50, delay=0, enableLearning=1,
                                   learningRule=lr)

c1 = net.createCompartment(prototype=p)
c2 = net.createCompartment(prototype=p)

conn1 = net.createConnection(c1, c2,

conn2 = net.createConnection(c1, c2,

PP = ProbeParameter
preTraceProbes1 = conn1.probe([PP.PRE_TRACE_X1, PP.PRE_TRACE_X2])
preTraceProbes2 = conn2.probe([PP.PRE_TRACE_X1])

plt.plot(preTraceProbes1[0].data, label="Syn1: X1", marker='o')
plt.plot(preTraceProbes1[1].data, label="Syn1: X2", marker='*')
plt.plot(preTraceProbes2[0].data, label="Syn2: X1", marker='o')

def conn_proto_with_learning_rule(self, net, learning_rule,
              x1Impulse=40, x1TimeConst=4, x2Impulse=20, x2TimeConst=16):
   lr = net.createLearningRule(
                              dw=learning_rule,
                              x1Impulse=x1Impulse,
                              x1TimeConstant=x1TimeConst,
                              x2Impulse=x2Impulse,
                              x2TimeConstant=x2TimeConst,
                              y1Impulse=40,
                              y1TimeConstant=4,
                              tEpoch=1)
   connProto = nx.ConnectionPrototype(
                              weight=50, delay=0,
                              enableLearning=1,
                              learningRule=lr)

   return connProto
