Compartment and CompartmentPrototype¶

Compartment Interface

class nxsdk.net.nodes.compartments.Compartment(nodeSet, nodeId)¶

Compartment Class

property activity¶

The activity corresponds to an exponential filter of a compartment's spiking history. For every compartment's spike, activity increases by activityImpulse and decays exponentially otherwise with a time constant activityTimeConstant. activity is only updated at intervals set by tEpoch within which only a single spike should be produced by the compartment otherwise the accuracy of activity with respect to an exact exponential filter degrades gracefully. The exponential decay itself is only approximated stochastically due to the limited bit precision. activity is used for threshold homeostasis but can also be used to approximate an instantaneous firing rate of a compartment for readout. (Default=0)

property activityImpulse¶

activitiyImpulse specifies the amount by which activity increases for each compartment spike. (Default=0)

property activityTimeConstant¶

Decay time constant by which activity decays approximately exponentially over time. (Default=0)

addPostTrace(postTrace)¶

Adds a new post trace to compartment. Cannot add more than 3 post traces per compartment.

property axonDelay¶

The compartment should only spike once per tEpoch if axonDelay > 0, otherwise additional delayed spikes will be lost and not get distributed.Axon delay has to satisfy the relationship 2*tEpoch >= axonDelay.``tEpoch`` can be up to 63. Therefore, axonDelay can have a maximum value of 189. (Default=0)

Type

There are two spike delay mechanism - discrete axonal and discrete synaptic delays. After a compartment spikes, the outgoing spike will be delayed by axonDelay before distribution to its outgoing synapses. Note

property axonId¶

axonId is the id of a compartment's output axon. (Default=-1)

property bias¶

Gets the bias current using the following computation \(bias = biasMant * 2^{biasExp}\). (Default=0)

property biasExp¶

The compartmentVoltage integrates a compartment's bias current which is represented by a mantissa and exponent according to \(bias = biasMant * 2^{biasExp}\) (Default=0)

property biasMant¶

The compartmentVoltage integrates a compartment's bias current which is represented by a mantissa and exponent according to \(bias = biasMant * 2^{biasExp}\) (Default=0)

property compartmentCurrent¶

The compartmentCurrent integrates incoming weighted spikes from the dendritic accumulator and possibly inputs from other compartments but decays exponentially otherwise according to: .. math:

u_i(t) = u_i(t-1) \cdot (2^{12} - \delta_i^{(u)})   \cdot2^{-12} +         2^{6+wgtExp}\sum_jw_{ij}\cdot s_j(t)

where \(\delta_i^{(u)}\) represents the decay and \(u_i(t)\) reprsents the compartment current at timestep t. For details see the Loihi Computational Model whitepaper. (Default=0)

property compartmentCurrentDecay¶

Decay constant by which the compartmentCurrent decays per time step. If tau is the decay time constant for the approximated exponential decay exp(-t/tau) time constant in time steps, then \(compartmentCurrentDecay = (1/tau)*2^{12}\). (Default=4096)

property compartmentCurrentTimeConstant¶

Gets compartmentCurrentTimeConstant (tau) which is the decay time constant for the approximated exponential current decay exp(-t/tau) in time steps. It is internally represented as \(compartmentCurrentDecay = (1/tau)*2^{12}\). (Default=1)

property compartmentJoinOperation¶

compartmentJoinOperation controls how the compartmentVoltage (V) or dendritic spikes (S) from input compartments get integrated with compartmentCurrent (U) into compartmentVoltage (V) of the current compartment: * SKIP (0) : Do not integrate input from input compartment. * ADD (1) : \(C.dV = C.U + A.V + B.V\) * MAX (2) : \(C.dV = max(C.U, A.V, B.V)\) * MIN (3) : \(C.dV = min(C.U, A.V, B.V)\) * PASS (4) : \(C.dV = A.S ? C.U + B.V : 0\) (Condition C.U + B.V integration by dendritic spike of A) * BLOCK (5) : \(C.dV = A.S ? 0: C.U + B.V\) (Condition C.U + B.V integration by A not providing a dendritic spike) * OR (6) : \(C.dS = A.S | B.S | C.S\) * AND (7) : \(C.dS = A.S \& B.S \& C.S\) (B.S=1 if stackIn==POP)

(Default=0)

property compartmentThreshold¶

Gets the voltage threshold. When a compartment reaches its voltage threshold, spikes are sent across all its connections. compartmentThreshold is internally represented as \(compartmentThreshold = vthMant * 2^{6}\). (Default=6400)

property compartmentVoltage¶

The compartmentVoltage integrates compartmentCurrent, the compartment bias and possibly inputs from other compartments according to: .. math:

compartmentVoltage(t) = compartmentVoltage(t-1) *

( 2^{12} - compartmentVoltageDecay )* 2^{-12} +

compartmentCurrent(t) + biasMant * 2^{biasExp}

For details see the Loihi Computational Model whitepaper. Note: When compartmentVoltage exceeds \(vthMant * 2^{6}\), the compartment produces a spike and compartmentVoltage resets. (Default=0)

property compartmentVoltageDecay¶

Decay constant by which the compartmentVoltage decays per time step. If tau is the decay time constant for the approximated exponential decay exp(-t/tau) time constant in time steps, then \(compartmentVoltageDecay = (1/tau)*2^{12}\). (Default=0)

property compartmentVoltageTimeConstant¶

Gets compartmentVoltageTimeConstant (tau) which is the decay time constant for the approximated exponential voltage decay exp(-t/tau) in time steps. It is internally represented as \(compartmentVoltageDecay = (1/tau)*2^{12}\). (Default=inf)

property enableHomeostasis¶

enableHomeostasis determines whether vThMant gets updated via activity driven threhold homeostasis. When enabled vThMant is modified in the following two cases: * activity > maxActivity : \(vThMant = homeostasisGain * (activity - maxActivity)\) * activity < minActivity : \(vThMant = homeostasisGain * (activity - minActivity)\)

Both variables can be randomized by adding \((LFSR() − 2^{7} + noiseMantAtCompartment * 2^{6}) * 2^{noiseExpAtCompartment − 7})\) where LFSR() returns an 8-bit random integer in the range 0..255. When spike back backpropagation is enabled, enableSpikeBackpropFrom selects the source that can give rise to a back propagating spike. (Default=0)

property functionalState¶

functionalState represents the current functional state of the compartment state machine. The possible states are: * INACTIVE (0) - Compartment does not get serviced * INHIBITED (1) - Compartment crossed \(-2^{vMinExp}\) and enableInhibition == True. Compartment is effectively INACTIVE until it receives excitatory stimulus again * IDLE (2) - Compartment gets serviced * FIRED_LAST_TIME_STEP (3) - Compartment has fired in the last time step * REFRACTORY (4) - The compartment has fired \(refractoryDelay > 1\) and the compartment has entered the refractory state in which it will not update compartmentVoltage * INHIBITED_REFRACTORY (5) - Mixture of INHIBITED and REFRACTORY. Compartment crossed \(-2^{vMinExp}\) but is not inhibited indefinitely but for duration of refractoryDelay

(Default=0)

property numDendriticAccumulators¶

There are two spike delay mechanism - shared axonal and discrete synaptic delays. After a spike gets generated and the axonal delay has passed, spikes get immediately distributed to the receiving synapses even with non-zero delay where the weight gets subsequently accumulated in the dendritic accumulator of a compartment. A compartment has either 8, 16, 32 or 64 dendritic accumulators which act as temporal buffers for a spike's weight scheduled to arrive at a future time step after a certain delay. numDendriticAccumulators must be larger than the maximum delay of all synapses connecting to a compartment. Specifically, the maximum synaptic delay < numDendriticAccumulators-1. It is recommended to select a small number and few different numbers to ease compilation of the network. The relationship between delay and numDendriticAccumulators is because one of these accumulators is associated with the current time step from which we read the current synaptic activation. No spike message should write to this current accumulator because that would lead to non-determinism of the readout depending on whether the asynchronous write or read happens first. This leaves numDendriticAccumulators-1 dendritic accumulators in which synaptic activations for future time steps can be accumulated. Any spike message has an implicit delay of 1 because a spike generated in the current time step can only be accumulated at the destination in the next time step. Thus, the synaptic delay must be <numDendriticAccumulators-1 or vice versa can be at most numDendriticAccumulators-2. (Default=8)

After a compartment spikes it enters a refractory state of duration refractoryDelay during which compartmentVoltage does not accumulate new inputs. The refractoryDelay can be randomized randomizeRefractoryDelay == True in which case noiseMantAtRefractoryDelay and noiseExpAtRefractoryDelay can be used to share the distribution of the noise. (Default=1)

property sharedFields¶
property stackIn¶

Multi-compartment neurons are implemented via a stack by which sending compartments can push the fields in stackOut to the stack and receiving compartments can pop or peek information from the stack according to stackIn setting. See compartmentJoinOperation on how the information from the sending compartment is integrated into the receiving compartment. The stack has a max depth of 2 so only binary trees can be built.

Note

To use stack operations, the functional state must be initialized as IDLE

Valid Enum values from nx.COMPARTMENT_INPUT_MODE are:

  • NOT PROVIDED (-1) - Auto Populated by Compiler

  • SKIP (0) - Don't read from the Compartment Stack

  • PEEK_A (1) - Peek from the Compartment Stack

  • POP_A (2) - Pop from the Compartment Stack

  • POP_A_AND_B (3) - Pop two from the Compartment Stack

(Default=-1)

property stackOut¶

Multi-compartment neurons are implemented via a stack by which sending compartments can push the fields in stackOut to the stack and receiving compartments can pop or peek information from the stack according to stackIn setting. See compartmentJoinOperation on how the information from the sending compartment is integrated into the receiving compartment. The stack has a max depth of 2 so only binary trees can be built.

Note

To use stack operations, the functional state must be initialized as IDLE

Valid Enum values from nx.COMPARTMENT_OUTPUT_MODE are:

  • NOT PROVIDED (-1) - Auto Populated by Compiler

  • SKIP (0) - Don't push to the Compartment Stack

  • PUSH (1) - Push (V,S) to the Compartment Stack. V denotes the membrane potential while S is the event when \(v > vth\)

(Default=-1)

property tEpoch¶

It is recommended to set refractoryDelay >= tEpoch if axonDelay > 0 to avoid loss of output spikes. Also it is recommended to use powers of 2 for efficiency. (Default=0)

Type

tEpoch controls the interval at which activity and vThMant (if enableHomeostasis == True) gets updated. Note

property thresholdBehavior¶

thresholdBehavior indicates what to do when compartmentVoltage exceeds \(vthMant * 2^{6}\): * SPIKE_AND_RESET (0) * SPIKE_AND_RANDOMIZE_REFRACTORY_DELAY (1) * NO_SPIKE_AND_PASS_V_LG_VTH_TO_PARENT (2) * NO_SPIKE_AND_PASS_V_LG_VTH_TO_PARENT_AND_SATURATE_V (3)

(Default=0)

property useDiscreteVTh¶

useDiscreteVTh determines whether the compartment threshold is stored as a shared parameter or discrete parameter. If may different compartment thresholds need to be configured, then enable useDiscreteVTh. Enabling useDiscreteVTh comes with a power and performance impact. (Default=0)

property vMaxExp¶

vMaxExp determines the maximum voltage vMax at which compartmentVoltage saturates \(vMax = 2^{vMaxExp}-1\). This can be used in non-spiking multi-compartment-neuron compartments to determine the maximum value of compartmentVoltage that gets passed on to the parent compartment in a dendritic tree. vMaxExp can only take value from (9,11,13,15,17,19,21,23). (Default=23)

property vMinExp¶

vMinExp determines the minimum voltage vMin at which compartmentVoltage saturates \(vMin = -2^{vMinExp}+1\). This can be used to limit how strongly a compartment can be inhibited or where it enters the INHIBITORY_REFRACTORY state. vMinExp can be any integer between 0 and 23. (Default=23)

property vThMant¶

When a compartment's compartmentVoltage exceeds a voltage threshold, a spike is produced and distributed by the compartment and distributed to other connected destinations while the compartmentVoltage resets. The threshold voltage is represented by a mantissa and a fixed exponent according to \(vth = vthMant * 2^{6}\) (Default=100)

After a spike gets generated and the axonal delay has passed, spikes get immediately distributed to the receiving synapses even with non-zero delay where the weight gets subsequently accumulated in the dendritic accumulator of a compartment. A compartment has either 8, 16, 32 or 64 dendritic accumulators which act as temporal buffers for a spike's weight scheduled to arrive at a future time step after a certain delay. numDendriticAccumulators must be larger than the maximum delay of all synapses connecting to a compartment. Specifically, the maximum synaptic delay < numDendriticAccumulators-1. It is recommended to select a small number and few different numbers to ease compilation of the network. The relationship between delay and numDendriticAccumulators is because one of these accumulators is associated with the current time step from which we read the current synaptic activation. No spike message should write to this current accumulator because that would lead to non-determinism of the readout depending on whether the asynchronous write or read happens first. This leaves numDendriticAccumulators-1 dendritic accumulators in which synaptic activations for future time steps can be accumulated. Any spike message has an implicit delay of 1 because a spike generated in the current time step can only be accumulated at the destination in the next time step. Thus, the synaptic delay must be <numDendriticAccumulators-1 or vice versa can be at most numDendriticAccumulators-2. (Default=8)

property powerGatingState¶

powerGatingState is a low level state that should normally not be touched by the user. (Default=3)

property randomizeCurrent¶

When True, randomizes compartmentCurrent. See enableNoise for details. randomizeCurrent and randomizeVoltage cannot be True at the same time. (Default=0)

property randomizeVoltage¶

When True, randomizes compartmentVoltage. See enableNoise for details. randomizeCurrent and randomizeVoltage cannot be True at the same time. (Default=0)

property refractoryDelay¶

After a compartment spikes it enters a refractory state of duration refractoryDelay during which compartmentVoltage does not accumulate new inputs. The refractoryDelay can be randomized randomizeRefractoryDelay == True in which case noiseMantAtRefractoryDelay and noiseExpAtRefractoryDelay can be used to share the distribution of the noise. (Default=1)

property stackIn¶

Multi-compartment neurons are implemented via a stack by which sending compartments can push the fields in stackOut to the stack and receiving compartments can pop or peek information from the stack according to stackIn setting. See compartmentJoinOperation on how the information from the sending compartment is integrated into the receiving compartment. The stack has a max depth of 2 so only binary trees can be built.

Note

To use stack operations, the functional state must be initialized as IDLE

Valid Enum values from nx.COMPARTMENT_INPUT_MODE are:

  • NOT PROVIDED (-1) - Auto Populated by Compiler

  • SKIP (0) - Don't read from the Compartment Stack

  • PEEK_A (1) - Peek from the Compartment Stack

  • POP_A (2) - Pop from the Compartment Stack

  • POP_A_AND_B (3) - Pop two from the Compartment Stack

(Default=-1)

property stackOut¶

Multi-compartment neurons are implemented via a stack by which sending compartments can push the fields in stackOut to the stack and receiving compartments can pop or peek information from the stack according to stackIn setting. See compartmentJoinOperation on how the information from the sending compartment is integrated into the receiving compartment. The stack has a max depth of 2 so only binary trees can be built.

Note

To use stack operations, the functional state must be initialized as IDLE

Valid Enum values from nx.COMPARTMENT_OUTPUT_MODE are:

  • NOT PROVIDED (-1) - Auto Populated by Compiler

  • SKIP (0) - Don't push to the Compartment Stack

  • PUSH (1) - Push (V,S) to the Compartment Stack. V denotes the membrane potential while S is the event when \(v > vth\)

(Default=-1)

property tEpoch¶

It is recommended to set refractoryDelay >= tEpoch if axonDelay > 0 to avoid loss of output spikes. Also it is recommended to use powers of 2 for efficiency. (Default=0)

Type

tEpoch controls the interval at which activity and vThMant (if enableHomeostasis == True) gets updated. Note

property thresholdBehavior¶

thresholdBehavior indicates what to do when compartmentVoltage exceeds \(vthMant * 2^{6}\): * SPIKE_AND_RESET (0) * SPIKE_AND_RANDOMIZE_REFRACTORY_DELAY (1) * NO_SPIKE_AND_PASS_V_LG_VTH_TO_PARENT (2) * NO_SPIKE_AND_PASS_V_LG_VTH_TO_PARENT_AND_SATURATE_V (3)

(Default=0)

property useDiscreteVTh¶

useDiscreteVTh determines whether the compartment threshold is stored as a shared parameter or discrete parameter. If may different compartment thresholds need to be configured, then enable useDiscreteVTh. Enabling useDiscreteVTh comes with a power and performance impact. (Default=0)

property vMaxExp¶

vMaxExp determines the maximum voltage vMax at which compartmentVoltage saturates \(vMax = 2^{vMaxExp}-1\). This can be used in non-spiking multi-compartment-neuron compartments to determine the maximum value of compartmentVoltage that gets passed on to the parent compartment in a dendritic tree. vMaxExp can only take value from (9,11,13,15,17,19,21,23). (Default=23)

property vMinExp¶

vMinExp determines the minimum voltage vMin at which compartmentVoltage saturates \(vMin = -2^{vMinExp}+1\). This can be used to limit how strongly a compartment can be inhibited or where it enters the INHIBITORY_REFRACTORY state. vMinExp can be any integer between 0 and 23. (Default=23)

property vThMant¶

When a compartment's compartmentVoltage exceeds a voltage threshold, a spike is produced and distributed by the compartment and distributed to other connected destinations while the compartmentVoltage resets. The threshold voltage is represented by a mantissa and a fixed exponent according to \(vth = vthMant * 2^{6}\) (Default=100)
