Connection and ConnectionPrototype¶

Connections

class nxsdk.net.nodes.connections.Connection(nodeSet, nodeId)¶

Connection is the base class for all different types of connections. To support connections between Compartments and Processes we need the following connection types:

1. Compartment -> Compartment: srcNode (outAx, inAx, Syn) dstNode
2. Process -> Compartment: srcNode ( , inAx, Syn) dstNode
3. Compartment -> Reinforcement: srcNode (outAx, inAx, ) dstNode
4. Process -> Reinforcement: srcNode ( , inAx, ) dstNode
5. Compartment -> Stub: srcNode (outAx, ) dstNode
6. Stub -> Compartment: srcNode ( , inAx, Syn) dstNode
7. Compartment -> Port: srcNode (outAx, , ) dstNode

Every Connection has a source and destination node but depending on the type it also holds references to an OutputAxon, InputAxon and Synapse. These three connection types are realized by the following subclasses:

1. ConnectionCC(ConnectionToCompartment)
2. ConnectionIC(ConnectionToCompartment)
3. ConnectionCR(ConnectionToCompartment)
4. ConnectionIR(ConnectionToCompartment)
5. ConnectionCS(ConnectionToCompartment)
6. ConnectionSC(ConnectionToCompartment)
7. ConnectionCP(ConnectionToCompartment)

property compressionMode¶

Configuring this parameter is only allowed until a network gets compiled because a change would require recompilation of the network.

Type

compressionMode determines how synapses are encoded in the synaptic memory. By default the compiler, will handle this configuration but by setting compressionMode to either SPARSE (0), RUNLENGTH (1) or DENSE (3), a certain encoding can be enforced. Note

property delay¶

delay allows to delay the accumulation of a spike in compartmentCurrent by a certain number of time steps if disableDelay == False. Otherwise delay acts as a general purpose synaptic variable like tag to be used for learning. The dynamic range of delay is 0..dd..62.But the precision 'dd' depends on numDelayBits \(dd=2^{6 - numDelayBits}\).

That means the actual delay per spike is :

  • \(numLsbBits = 6 - numDelayBits\)

  • \(actDelay=(delay >> numLsbBits) << numLsbBits\)

This corresponds to only preserving the numDelayBits most significant bits. Note: When learning is enabled, delay will saturate at the dynamic range boundaries. Also note that numDendriticAccumulator of any compartment connected by a synapse with nonzero delay must be set to > delay`+1 to support these synapses. See description for ``numDendriticAccumulator` for additional details. Note: If the value of delay is less than or equal to 7, the compiler will calculate 'numDelayBits' and numDendriticAccumulator automatically. If the value of delay is greater than 7, please manually set compartment 'numDendriticAccumulators' and connection 'numDelayBits'.

property disableDelay¶

Disables synaptic delay of synapse. If disableDelay == True, then delay is not used as a synaptic delay but a general purpose variable for learning only.

property postSynResponseMode¶

postSynResponseMode determines the post synaptic response behavior of a synapse.

If set to EXPONENTIAL (0), the weight will be accumulated in compartmentCurrent and decay exponentially.

If set to BOX (1), the weight will be added to compartmentCurrent but will not decay until the synaptic delay period has passed. After the delay period has passed, weight will be subtracted from compartmentCurrent and hence the post synaptic response resembles a box stimulus. Note: This requires \(delay > 0\) and disableDelay == False, otherwise no effective weight will be accumulated in compartmentCurrent.

property weightExponent¶

Before the weight is accumulated in compartmentCurrent, an additional exponential scaling by \(2^{weightExponent}\) is performed.

property weightLimitMant¶

The synaptic weight can be saturated at a certain limit before accumulation in compartmentCurrent when weightLimitMant > 0. This is useful to limit the strength of a synaptic weight to a certain range during spike accumulation but to allow a learning rule to increase the weight beyond this limit as a means to realize a synaptic permanence or a resistance of a weight to be unlearned.When activated, weigthLimitMant is the mantissa of the weight limit \(wgtLim = weightLimitMant * 2^{weightLimitExp}\) at which the weight saturates before accumulation in compartmentCurrent.

The accumulated weight is given by: \(accWgt = sign(weight) * \min\limits_{wgtLim, |weight| * 2^{8 - (numWeightBits - IS\_MIXED) + weightExponent}}\)

property weigthLimitExp¶

weightLimitExp is the exponent of the weight limit at which a synaptic weight saturates before accumulation in compartmentCurrent. See weightLimitMant for details.

class nxsdk.net.nodes.connections.Synapse(nodeSet, nodeId)¶

A Synapse connects an InputAxon to a Compartment. Synapses have three dynamic state variables: weight, delay and tag. The weight is accumulated as a result of an incoming spike from the InputAxon in the dendriticAccumulators of the post synaptic compartment. ddendriticAccumulators acts as a (circular) temporal buffer for spikes scheduled to arrive at future time steps. If the synapse delay is nonzero, an incoming spike will be immediately accumulated but in a dendriticAccumulator for a future time step corresponding to the delay of the synapse. When that delay has passed, all weights accumulated so far in that dendriticAccumulator will be accumulated in the compartmentCurrent state variable. LearningRules can be assigned to synapses in which case a synapse can be associated with up to two different preTraces, up to three different postTraces and one rewardTrace. Besides weight and delay, a synapse has a tag state which is only available as a general purpose variable for learning. In addition, the delay variable can be turned into another general purpose variable by setting Synapse.disableDelay == True.

If set to BOX (1), the weight will be added to compartmentCurrent but will not decay until the synaptic delay period has passed. After the delay period has passed, weight will be subtracted from compartmentCurrent and hence the post synaptic response resembles a box stimulus. Note: This requires \(delay > 0\) and disableDelay == False, otherwise no effective weight will be accumulated in compartmentCurrent. (Default=0)

property weightLimitMant¶

The synaptic weight can be saturated at a certain limit before accumulation in compartmentCurrent when weightLimitMant > 0. This is useful to limit the strength of a synaptic weight to a certain range during spike accumulation but to allow a learning rule to increase the weight beyond this limit as a means to realize a synaptic permanence or a resistance of a weight to be unlearned.When activated, weigthLimitMant is the mantissa of the weight limit \(wgtLim = weightLimitMant * 2^{weightLimitExp}\) at which the weight saturates before accumulation in compartmentCurrent.

property delay¶

delay allows to delay the accumulation of a spike in compartmentCurrent by a certain number of time steps if disableDelay == False. Otherwise delay acts as a general purpose synaptic variable like tag to be used for learning. The dynamic range of delay is 0..dd..62.But the precision 'dd' depends on numDelayBits \(dd=2^{6 - numDelayBits}\).

That means the actual delay per spike is :

  • \(numLsbBits = 6 - numDelayBits\)

  • \(actDelay=(delay >> numLsbBits) << numLsbBits\)

This corresponds to only preserving the numDelayBits most significant bits. Note: When learning is enabled, delay will saturate at the dynamic range boundaries. Also note that numDendriticAccumulator of any compartment connected by a synapse with nonzero delay must be set to > delay`+1 to support these synapses. See description for ``numDendriticAccumulator` for additional details. Note: If the value of delay is less than or equal to 7, the compiler will calculate 'numDelayBits' and numDendriticAccumulator automatically.
