property bapActionÂ¶

(uint2) Specifies action to apply when a back-propagating action potential occurs. Each bit independently activates one or more possible actions:

  • Bit 0: Activate refractory phase for duration specified in CxProfileCfg.refractDelay.

  • Bit 1: Propagate action potential back from compartment to its synaptic post traces for learning.

property bapSrcÂ¶

(uint4) Specifies which events can trigger a back-propagating action potential (bAP). Each bit independently activates one or more possible sources:

  • Bit 0: Activate bAP when compartment voltage exceeds threshold, i.e. \(CxState.v > VthProfileCfg[CxCfg.vthProfile].vth\).

  • Bit 1: Unused.

  • Bit 2: Activate bAP when input compartment 0 as defined by multi-compartment stack spikes. I.e. when \(A.S=1\).

  • Bit 3: Activate bAP when input compartment 1 as defined by multi-compartment stack spikes. I.e. when \(B.S=1\).

property joinOpÂ¶

(uint3) Specifies operation how to join input from child compartments with this associated compartment using this profile via multi-compartment stack.

  • SKIP (0): Don't integrate input from stack.

  • ADD (1): Add input from child compartment \(C.dV = C.U + A.V + \ B.V\).

  • MAX (2): Use max of values on stack and this compartment's compartment current \(C.dV = max(C.U, A.V, B.V)\).

  • MIN (3): Use min of values on stack and this compartment's compartment current \(C.dV = min(C.U, A.V, B.V)\).

  • PASS (4): Let this compartment's compartment current pass if child compartments spiked \(C.dV = A.S ? C.U + B.V : 0\).

  • BLOCK (5): Block this compartment's compartment current if child compartment has spiked \(C.dV = A.S ? 0: C.U + B.V\).

  • OR (6): Generate output spike if this compartment or any of its child compartments has spiked \(C.dS = A.S | B.S | C.S\).

  • AND (7): Generate output spike only if this compartment and all of

its child compartments have spiked \(C.dS = A.S \& B.S \& C.S\)..

property refractDelayÂ¶

(uint 6) Specifies duration of refractory phase during which compartment voltage is clamped to 0 and neuron does not spike. Note: During refractory phase CxState[j].v is overloaded by the refractory delay and refractory counter. This does not represent the actual membrane potential.

property stackInÂ¶

(uint2) Specifies what information gets retrieved from multi-compartment stack.

  • SKIP (0): Don't read from stack.

  • PEEK_A (1): Peek from stack and assign to A.

  • POP_A (2): Pop top element from stack and assign to A.

  • POP_A_AND_B (3): Pop both first and second element from stack and assign to A and B.

property stackOutÂ¶

(uint2) Specifies what information gets pushed onto multi-compartment stack by this compartment.

  • SKIP (0): Don't push anything to stack.

  • PUSH (1): Push (V,S) to the Compartment Stack. V denotes the membrane potential while S is the event when \(v > vth\).

property threshOpÂ¶

(uint2) Specifies operation to apply then compartment voltage exceeds threshold, i.e. \(CxState.v > VthProfileCfg[CxCfg.vthProfile].vth\).

  • SPIKE_AND_RESET (0):Generate output spike and reset compartment voltage to 0.

  • SPIKE_AND_RANDOMIZE_REFRACTORY_DELAY (1): Generate output spike, enter refractory phase but add random number to refractory delay.

  • NO_SPIKE_AND_EVALUATE_VOLTAGE (2): Generate no spike and push \(S = CxState.v > VthProfileCfg[CxCfg.vthProfile].vth\) onto multi-compartment stack.

  • NO_SPIKE_AND_EVALUATE_E (3): Generate no spike, push \(S = CxState.v > VthProfileCfg[CxCfg.vthProfile].vth\) onto multi-compartment stack and saturate compartment voltage at threshold.
